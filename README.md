# Vantrex: Context-Aware IoT Cyber Activity Detection

**Capstone Project — Binary Classification of IoT Network Traffic**

Vantrex is a machine learning system that analyzes IoT network traffic features to detect whether a device is behaving normally or showing signs of hidden cyber activity.

---

## Dataset

**TON-IoT Network Dataset** — `data/train_test_network.csv`  
- 211,043 rows, 44 columns  
- Binary target: `label` (0 = Normal, 1 = Attack)  
- ~76% Attack, ~24% Normal (imbalanced — F1-score is primary metric)

## Step 1 — Run the Notebook

Open `notebook.ipynb` in Jupyter or VS Code and run all cells from top to bottom:

```bash
jupyter notebook notebook.ipynb
```

The notebook will:
1. Load and clean the TON-IoT dataset
2. Perform exploratory data analysis (EDA) and save figures to `figures/`
3. Train Logistic Regression, Decision Tree, and Random Forest models
4. Build and train a Keras Dense Neural Network
5. Compare all models by F1-Score
6. Save the best model to `models/best_pipeline.pkl`
7. Save the NN to `models/nn_model.keras`
8. Save results to `results/model_comparison.csv`

**Important:** The notebook must be run before launching the app. The app requires the saved model files.

---

## Step 2 — Run the Streamlit Website

```bash
streamlit run app/app.py
```

Then open `http://localhost:8501` in your browser.

**Website features:**
- **Manual Input tab** — enter individual network traffic feature values and get a prediction
- **CSV Batch Upload tab** — upload a CSV file with multiple records for batch prediction
- **🤖 AI Assistant tab** — rule-based NLP chatbot that answers questions about the model, metrics, risk levels, and how to use the app (fully offline, no external API)
- Output for each prediction:
  - Prediction: Normal or Attack
  - Confidence: attack probability as a percentage
  - Risk Level: Low / Medium / High
- Neural Network toggle in the sidebar (optional)
- Educational disclaimer

## Models

| File | Description |
|---|---|
| `models/best_pipeline.pkl` | sklearn Pipeline bundling ColumnTransformer + best classifier |
| `models/nn_model.keras` | Keras Sequential Dense Neural Network |
| `models/nn_preprocessor.pkl` | ColumnTransformer fitted for NN input |
| `models/label_mapping.json` | Maps 0 → Normal, 1 → Attack |
| `models/feature_config.json` | Lists of numeric and categorical feature names |

---

## Models Trained

| Model | Type |
|---|---|
| Logistic Regression | scikit-learn, linear baseline |
| Decision Tree | scikit-learn, max_depth=15 |
| Random Forest | scikit-learn, 100 estimators |
| Keras Dense NN | TensorFlow/Keras, 3 hidden layers + Dropout |

Primary evaluation metric: **F1-Score** (chosen because the dataset is imbalanced ~76% Attack).
